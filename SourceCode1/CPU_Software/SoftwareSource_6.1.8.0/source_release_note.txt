INSTALLATION INSTRUCTIONS 
-------------------------
Perform the following steps to install the software onto the target platform:

	Unzip the archive file.

	Copy the folders "c" and "d" from the archive onto a USB key.

	On the target hardware disable the Windows Embedded File System
	Write Filters.

	Reboot the hardware after disabling the write filters.

	Copy the contents of the folder "c" from the USB key to c:\ on
	the target hardware.

	Copy the folder "d" the folder d from the USB key to d:\ on the
	target hardware.

		The C:\ drive of the target system should now contain an
		"AMOT" folder

		The D:\drive of the target system should now contain the
		following folders:
			
			config
			database
			exports
			log
			reports
			updates

	Create a Shortcut to c:\AMOT\Launcher.exe.

	Copy the Shortcut to the startup Folder on the Start menu.

	Reboot the target hardware.

	Using a Web Browser open the page <target hardware ip>/config.

	Navigate to Advanced Settings | Write Filter Settings.

		Verify that the Write Filters are disabled.

	Navigate to Serial Configuration.

		Setup the correct Serial Configuration for the SPU Port.

		Setup the correct Serial Configuration for the External
		Port (Used to access the Modbus Server).

	Navigate to Network Configuration.

		If required configure the network ports for the target
		network.
	
	Navigate to Advanced Settings | Write Filter Settings.

		Enable the Write Filters.
		Reboot the system.

	The system is now ready to use.

	To decrease the number of requests to the SPU during operation
	the SPUs configuration should be setup using the SPU Settings
	Configuration Web Page located at <target hardware ip>/config.
	You may need to import a SPU configuration file before settings
	can be changed (See samples folder).

Please note before editing Serial Configuration or Network Configuration
the Write Filters should be disabled using the Advanced Settings | Write
Filter Settings or the Write Filters Tray Icon application on the target
hardware.

will need to be deleted before upgrading to this release.

Software update packages may also not work with previous versions.


NOTES
-----
Please take care when editing the Modbus map file.  Internal software
components of the system rely on the Modbus Server exposing registers at
the addresses defined in the register.map file included with this
release.  If a new range is required for use by 3rd party software
please define this range outside of the ranges defined in the file.

All software components in the system can be switched from using the
mapped version of the Modbus Server registers to using the non-mapped
versions by editing the following application configuration files:

	c:\AMOT\Logger\Framework.exe.config
	c:\AMOT\Website\Web.config

	In each file locate the <DataCollectionSDK> XML element and
	change the <Modbus> child element from:

	<Modbus ModbushostIP="127.0.0.1"  ModbusExternalHostPort="502"
	ModbusInternalHostPort="1502" ModbusTCPIPTimeout="30000"
	ModbusSlaveUnitId="1" PreferInternal="false"/>

	to

	<Modbus ModbushostIP="127.0.0.1"  ModbusExternalHostPort="1502"
	ModbusInternalHostPort="1502" ModbusTCPIPTimeout="30000"
	ModbusSlaveUnitId="1" PreferInternal="true"/>

Utilities
---------
This section describes how to use the various utilities that are
included with this release.

  ConvertDatabase
  ---------------
The ConvertDatabase utility is used to convert databases from SQLExpress
to SQLite.  The utility is designed to be run on a XTS-W+ V1.5 system
before upgrading it to this version of the software.  The utility will
convert all the data from the V1.5 database and store it in a SQLite
format database for using on a upgraded system.  When running the
application the user should select an output folder for the converted
database and click the convert button.  The utility will create two
database files in the selected folder:

	* amot_xtsii_data.db3

	* amot_xtsii_event.db3

These files should be copied to d:\databases once the system has been
upgraded.  Please be aware that these database files may be very large.
An external USB hard drive may be required to copy them from the old
to the new system.

  PasswordResetResponse
  ---------------------
The PasswordResetResponse utility is used to generate the code that is
sent to a user when they have selected to reset the users passwords.  To
 use the utility open a command line and enter:

	PasswordResetResponse <code from user>

The utility will then output the response code that should be sent to
the user on the command line.
